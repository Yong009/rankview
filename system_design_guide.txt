# RankView 프로젝트 설계 및 수정 가이드 (System Design & Modification Guide)

## 1. 시스템 아키텍처 개요
본 프로젝트는 Spring Boot 기반의 백엔드와 Vanilla JS 기반의 프론트엔드로 구성된 키워드 순위 추적 관리 시스템입니다.

- Backend: Java 17+, Spring Boot 3.x
- Frontend: HTML5, CSS3, JavaScript (JSP 지원 포함)
- Database: AWS Lightsail MySQL
- External API: Naver Search API
- Automation/Utility: Python scripts, Playwright (순위 체크용)

---

## 2. 프로젝트 디렉토리 구조 및 역할

### [Backend] src/main/java/com/example/rankview
- controller: API 엔드포인트 정의 (REST API)
  * 수정 시나리오: 새로운 API 추가, 요청 주소 변경, 파라미터 수정
- service: 비즈니스 로직 처리 (순위 업데이트, 엑셀 처리)
  * 수정 시나리오: 순위 계산 로직 변경, Naver API 연동 로직, 엑셀 파싱 로직 수정
- entity: 데이터베이스 테이블 구조 정의
  * 수정 시나리오: DB 컬럼 추가/삭제, 데이터 모델 변경
- repository: DB 접근 인터페이스 (JPA)
  * 수정 시나리오: 새로운 DB 쿼리(검색, 필터링) 추가
- config: 시스템 설정 (Security, Web 등)
  * 수정 시나리오: 권한 설정, CORS 해결, 빈(Bean) 등록

### [Frontend] Root Directory
- index.html: 메인 대시보드 구조
  * 수정 시나리오: 버튼 추가, 테이블 레이아웃 변경, 모달 창 구조 수정
- app.js: UI 렌더링 및 백엔드 API 통신
  * 수정 시나리오: 버튼 클릭 이벤트, API 호출 방식, 데이터 화면 표시 로직 수정
- style.css: 디자인 및 스타일링
  * 수정 시나리오: 색상, 폰트, 반응형 레이아웃, 애니메이션 수정

### [Resources] src/main/resources
- application.properties: 시스템 설정 및 DB 접속 정보
  * 수정 시나리오: DB 주소/비밀번호 변경, Naver API Key 변경, 서버 포트 수정

---

## 3. 핵심 엔티티 및 DB 테이블 구성
1. User: 사용자 정보 및 로그인 권한 관리
2. Folder: 키워드를 그룹화하는 폴더 (트리 구조 지원)
3. KeywordRank: 추적 대상 키워드 정보 (MID, 키워드명, 타겟 순위 등)
4. KeywordDailyData: 일자별 순위 이력 데이터 (통계 및 그래프용)

---

## 4. 기능별 수정 가이드 (Quick Fix)

### 순위 체크/업데이트 로직을 고치고 싶다면?
- RankUpdateService.java: 실제 순위를 가져오는 핵심 로직
- PlaywrightService.java: 브라우저 자동화(Naver 순위 체크 등) 로직
- KeywordController.java: 순위 업데이트 API (/api/keywords/{id}/update)

### 엑셀 업로드/다운로드 기능을 고치고 싶다면?
- ExcelController.java: 엑셀 관련 API 엔드포인트
- ExcelService.java: 엑셀 파일 읽기/쓰기 로직
- debug_excel.py: 엑셀 파일 형식 테스트용 파이썬 스크립트

### 화면의 디자인이나 동작을 고치고 싶다면?
- UI 디자인: style.css에서 관련 클래스 수정
- 버튼 클릭 시 동작: app.js의 setupEventListeners() 함수에서 해당 버튼 ID 확인
- 데이터 표시: app.js의 renderKeywords() 함수에서 테이블 생성 로직 수정

### 데이터베이스를 직접 관리하거나 초기화하고 싶다면?
- update_db.py: DB 테이블 최신화 또는 데이터 보정
- check_tables.py: DB 테이블 정상 생성 여부 확인

---

## 5. 작업 시 주의사항
1. DB 변경 시: Entity 수정 후 ddl-auto=update에 의해 자동 반영되나, 중요한 변경은 수동 확인 필요
2. API 호출: app.js의 모든 호출은 세션 기반 인증이 필요함 (로그인 상태 필수)
